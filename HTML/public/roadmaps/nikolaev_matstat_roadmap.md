### Лекция 1: Введение. Задача игрового автомата. Модели данных.

**Приоритет:** 🟡 Средняя

Николаев ставит задачу: журналист имеет 138 игр на автомате, средний выигрыш 0.38, компания заявляет 0.92. Можем ли утверждать, что автомат жульничает? На этом примере вводятся: случайные величины, независимые реализации, статистические модели (4 модели разной сложности), эмпирическое распределение.

**Основные темы:**
- Постановка статистической задачи на реальном примере
- Случайные величины и независимые реализации (i.i.d.)
- Эмпирическое распределение (модель 1): случайный элемент выборки с равными вероятностями
- Полная модель (модель 2): все 343 комбинации с отдельными вероятностями
- Модель с независимостью окошек (модель 3): 21 параметр вместо 343
- Модель с одинаковыми распределениями (модель 4): 7 параметров
- Компромисс между сложностью модели и количеством параметров

**Ключевые концепции и формулы:**
- Эмпирическая случайная величина: P(x) = (кратность x) / n
- Выборочное среднее как мат. ожидание эмпирического распределения
- Независимость: P(комбинация) = P(окошко1) * P(окошко2) * P(окошко3)

**Для MLE собеседований:**
| Тема | Статус | Комментарий |
|------|--------|-------------|
| i.i.d. предположение | 🟢 | «Что такое i.i.d.? Почему это важно?» |
| Эмпирическое распределение | 🟡 | Фундамент для bootstrap |
| Параметрические vs непараметрические модели | 🟡 | Общее понимание |
| Баланс сложности модели | 🟢 | Связь с bias-variance tradeoff |

**Что вынести:** Статистика начинается с модели. Данные без модели бесполезны. Чем больше предположений — тем меньше параметров, но тем больше шансов ошибиться.

**Зависимости:** Нет (первая лекция).

### Лекция 2: Модели реальности. Bootstrap-идея. Метод максимального правдоподобия.

**Приоритет:** 🟢 Высокая

Продолжение задачи с автоматом. Ключевая идея: чтобы изучить модель, нужна «модель реальности». Если реальность недоступна — заменяем её другой моделью. Генерация данных из модели, сравнение моделей 1 и 3. Введение метода максимального правдоподобия (MLE).

**Основные темы:**
- Модель реальности vs изучаемая модель
- Генерация выборок из модели (ресэмплинг, bootstrap-идея)
- Распределение выборочного среднего при разных моделях реальности
- Метод максимального правдоподобия (MLE): определение, идея
- Правдоподобие = произведение вероятностей (для i.i.d.)
- MLE для модели 3: оценки = частоты

**Ключевые концепции и формулы:**
- Правдоподобие: L(m) = P(data | model = m) = prod P(xi | m)
- MLE: argmax_m L(m)
- Для модели 3: p_hat(картинка, окошко) = (кратность) / n
- Логарифм правдоподобия: log L = sum n_ij * log p_ij

**Для MLE собеседований:**
| Тема | Статус | Комментарий |
|------|--------|-------------|
| Метод максимального правдоподобия (MLE) | 🟢 | Топ-вопрос: «Что такое MLE? Как связан с cross-entropy?» |
| Правдоподобие для i.i.d. данных | 🟢 | Произведение -> сумма логарифмов |
| Модель реальности как инструмент | 🟡 | Понимание bootstrap |
| Выразительность модели | 🟡 | Более сложная модель может генерировать невиданные данные |

**Что вынести:** MLE — выбираем модель, при которой наши данные наиболее правдоподобны. Для i.i.d. данных правдоподобие = произведение. MLE-оценки часто совпадают с частотами.

**Зависимости:** Лекция 1.

### Лекция 3: Правдоподобие. MLE вывод. Проверка гипотез (введение).

**Приоритет:** 🟢 **КРИТИЧЕСКАЯ**

Завершение задачи с автоматом. Формальный вывод MLE. Введение проверки гипотез: H0 vs H1, частотный vs байесовский подход, отношение правдоподобия (likelihood ratio). Разница между «правдоподобно» и «вероятно».

**Основные темы:**
- Вывод MLE для модели 3: максимизация правдоподобия аналитически
- Логарифм отношения правдоподобия как функция потерь
- Нулевая гипотеза (H0) и альтернатива (H1)
- Частотный подход: предполагаем H0, смотрим правдоподобие данных
- Байесовский подход: оцениваем вероятность гипотезы
- Правдоподобие != истинность (ключевое различие!)
- Проблема составной гипотезы

**Ключевые концепции и формулы:**
- MLE для мультиномиального распределения: p_hat_j = n_j / n
- Отношение правдоподобия: Lambda = L(H0) / L(H_all)
- -log(Lambda) как мера отклонения от H0
- Кризис воспроизводимости: правдоподобие не равно истинности

**Для MLE собеседований:**
| Тема | Статус | Комментарий |
|------|--------|-------------|
| Likelihood ratio test | 🟢 | Фундамент для A/B тестов |
| Частотный vs байесовский подход | 🟢 | «В чём разница?» — стандартный вопрос |
| MLE вывод | 🟢 | «Выведите MLE для нормального распределения» |
| Правдоподобие != вероятность | 🟢 | Частая ловушка на собесах |

**Что вынести:** Отношение правдоподобия — универсальный инструмент сравнения моделей. Частотный подход: «если H0 верна, насколько правдоподобны данные?». Байесовский: «какова вероятность H0 при данных данных?». Это РАЗНЫЕ вопросы.

**Зависимости:** Лекции 1-2.

### Лекция 4: Проверка гипотез: принципы, функции потерь, альтернативы.

**Приоритет:** 🟢 **КРИТИЧЕСКАЯ**

Детальный разбор механизма проверки гипотез. Альтернатива определяет, какие значения «экстремальны». Функция потерь как «линейка» расстояния от H0. Проверка гипотезы однородности (одинаковы ли распределения в окошках). Критерий отношения правдоподобия для составных гипотез.

**Основные темы:**
- H0 vs H1: альтернатива задаёт направление «экстремальности»
- Метафора: H0 — синяя машина; H1 — жёлтая / внедорожник / собака → разные «функции потерь»
- Функция потерь = рулетка, мерящая расстояние от H0 в сторону H1
- Распределение функции потерь при H0 → что «много», что «мало»
- Гипотеза однородности: одинаковы ли распределения в окошках?
- Критерий отношения правдоподобия для проверки однородности

**Ключевые концепции и формулы:**
- Экстремальные значения определяются альтернативой
- P(данные | H0) → распределение статистики при H0
- Если наше значение «слишком экстремально» при H0 → отвергаем H0
- Сравнение вложенных моделей через likelihood ratio

**Для MLE собеседований:**
| Тема | Статус | Комментарий |
|------|--------|-------------|
| Проверка гипотез: логика | 🟢 | Фундамент A/B тестирования |
| Функция потерь / статистика критерия | 🟢 | Выбор правильной метрики |
| Однородность выборок | 🟡 | Полезно для feature engineering |
| Вложенные модели | 🟡 | Model selection |

**Что вынести:** Альтернатива задаёт, что считать «экстремальным». Одни и те же данные могут быть экстремальными для одной альтернативы и нормальными для другой. Функция потерь — не абстракция, а инструмент.

**Зависимости:** Лекции 1-3.

### Лекция 5: Выборочные характеристики. Описательная статистика.

**Приоритет:** 🟢 Высокая

Формализация: выборка, эмпирическое распределение, теорема Гливенко-Кантелли, теорема Колмогорова. Выборочные характеристики: среднее, дисперсия (обычная и исправленная), моменты, квантили, квартили, медиана. Характеристики положения и разброса. Коэффициент асимметрии и эксцесс.

**Основные темы:**
- Формальное определение выборки и эмпирического распределения
- Теорема Гливенко-Кантелли: sup|Fn(x) - F(x)| → 0 п.н.
- Теорема Колмогорова: скорость сходимости ~1/sqrt(n)
- Выборочное среднее = мат. ожидание эмпирического распределения
- Выборочная дисперсия (1/n) vs исправленная дисперсия (1/(n-1))
- Квантили, квартили, медиана
- Вариационный и статистический ряды
- Характеристики положения (среднее, медиана, мода) и разброса (размах, IQR, std)
- Коэффициент асимметрии (skewness) и эксцесс (kurtosis)

**Ключевые концепции и формулы:**
- X_bar = (1/n) * sum(Xi)
- S^2 = (1/n) * sum(Xi - X_bar)^2 (выборочная)
- S^2_corrected = (1/(n-1)) * sum(Xi - X_bar)^2 (исправленная)
- Skewness = E[(X-mu)^3] / sigma^3
- Kurtosis = E[(X-mu)^4] / sigma^4 - 3

**Для MLE собеседований:**
| Тема | Статус | Комментарий |
|------|--------|-------------|
| Выборочное среднее и дисперсия | 🟢 | Базовые знания, спрашивают всегда |
| Почему делим на n-1 (Бессель) | 🟢 | Классический вопрос: «Почему n-1?» |
| Квантили и медиана | 🟢 | Используются в метриках, робастной статистике |
| Skewness и kurtosis | 🟡 | Полезно для EDA, feature engineering |
| Теорема Гливенко-Кантелли | 🟡 | Теоретическое обоснование bootstrap |

**Что вынести:** Выборочные характеристики — это просто характеристики эмпирического распределения. Исправленная дисперсия (n-1) даёт несмещённую оценку. Коэффициент асимметрии и эксцесс показывают «форму» распределения.

**Зависимости:** Лекции 1-2 (эмпирическое распределение).

### Лекция 6: Метод Монте-Карло. ЦПТ. Importance sampling. Байесовский подход.

**Приоритет:** 🟢 **КРИТИЧЕСКАЯ**

Метод Монте-Карло для оценки мат. ожиданий. Центральная предельная теорема (ЦПТ) и погрешность ~1/sqrt(n). Правило трёх сигм. Оценка вероятностей событий. Importance sampling. Введение в байесовский подход: априорное, апостериорное распределение, шансы.

**Основные темы:**
- Метод Монте-Карло: оценка E[f(X)] через среднее по выборке
- ЦПТ: (X_bar - mu) * sqrt(n) → N(0, sigma^2)
- Погрешность MC: ~sigma/sqrt(n), правило 3 сигм
- Оценка вероятностей через индикаторные функции
- Оценка дисперсии через E[X^2] - (E[X])^2
- Importance sampling: замена распределения для снижения дисперсии
- Байесовский подход: prior * likelihood ∝ posterior
- Априорные и апостериорные шансы
- Пример с интересной болезнью: P(болен|+) через шансы

**Ключевые концепции и формулы:**
- MC оценка: E[f(X)] ≈ (1/n) * sum f(Xi)
- Погрешность: O(1/sqrt(n)) — для 10x точности нужно 100x данных
- Importance sampling: E_p[f(X)] = E_q[f(X) * p(X)/q(X)]
- Апостериорные шансы = априорные шансы * отношение правдоподобия
- Пример: P(болен) = 0.001, тест 99% → P(болен|+) ≈ 9%

**Для MLE собеседований:**
| Тема | Статус | Комментарий |
|------|--------|-------------|
| Метод Монте-Карло | 🟢 | Используется повсеместно в ML |
| ЦПТ и сходимость 1/sqrt(n) | 🟢 | «Почему больше данных лучше?» |
| Importance sampling | 🟢 | Используется в RL (PPO), VAE |
| Байесовский подход: prior * likelihood ∝ posterior | 🟢 | Топ-вопрос: «Что такое теорема Байеса?» |
| Пример с болезнью (base rate fallacy) | 🟢 | Классический пример на собесах |

**Что вынести:** MC — универсальный инструмент. ЦПТ даёт погрешность ~1/sqrt(n). Importance sampling позволяет считать интегралы, даже если не умеем генерировать из нужного распределения. Байес: апостериорные шансы = априорные * likelihood ratio.

**Зависимости:** Лекция 5 (выборочные характеристики).

### Лекция 7: Байесовский классификатор. Оптимальное правило. Средний риск.

**Приоритет:** 🟢 **КРИТИЧЕСКАЯ**

Теорема об оптимальном байесовском классификаторе. Средний риск, функция потерь для классификации. Частный случай: 0-1 loss → максимум апостериорной вероятности (MAP). Примеры: медицина, суд. Априорные и апостериорные вероятности. Шансы как удобный инструмент вычислений.

**Основные темы:**
- Средний риск классификатора: R(a) = sum lambda(y, y') * P(y|x) * P(x)
- Теорема: оптимальный классификатор минимизирует средний риск для каждого x
- Частный случай: 0-1 loss → MAP (maximum a posteriori)
- Частный случай: штраф зависит только от истинного класса → взвешенный MAP
- Примеры с разными штрафами: медицина (пропустить болезнь >> ложная тревога), суд (осудить невиновного >> отпустить виновного)
- Связь с формулой Байеса: P(y|x) ∝ P(x|y) * P(y)
- Вычисления через шансы: удобство для нескольких гипотез

**Ключевые концепции и формулы:**
- Средний риск: R = sum_y sum_y' lambda(y,y') * P(y|x) * I[a(x)=y']
- Оптимальный классификатор: a*(x) = argmin_y' sum_y lambda(y,y') * P(y|x)
- При 0-1 loss: a*(x) = argmax_y P(y|x) = argmax_y P(x|y)*P(y)
- Апостериорные шансы = априорные шансы * отношение правдоподобия

**Для MLE собеседований:**
| Тема | Статус | Комментарий |
|------|--------|-------------|
| Байесовский оптимальный классификатор | 🟢 | «Как устроен Naive Bayes?» |
| MAP-решение | 🟢 | «Связь MLE и MAP?» — стандартный вопрос |
| Неравные штрафы за ошибки | 🟢 | Class imbalance, cost-sensitive learning |
| Априорные/апостериорные вероятности | 🟢 | Фундамент байесовского ML |
| P(y|x) ∝ P(x|y) * P(y) | 🟢 | Naive Bayes, generative models |

**Что вынести:** Оптимальный классификатор = минимум среднего штрафа для каждого x. При равных штрафах → MAP. Неравные штрафы кардинально меняют решение (медицина, суд). Шансы — удобный способ вычислений с Байесом.

**Зависимости:** Лекция 6 (байесовский подход).

### Лекция 8: Частотный подход. Оценки параметров. Доверительные интервалы.

**Приоритет:** 🟢 Высокая

Частотный подход: вероятность есть только у повторяемых экспериментов. Нет априорных распределений. Точечные и интервальные оценки. Статистика = функция от данных. Оценка как случайная величина. Анализ оценок: смещение, разброс. Доверительные интервалы: ширина, покрытие, разрешающая способность.

**Основные темы:**
- Частотный подход: отказ от априорных распределений
- Вероятность существует только у повторяемых событий
- Параметр = функционал от распределения, оценка = статистика
- Оценка как случайная величина (estimator vs estimate)
- Свойства оценок: смещение (bias), разброс (variance)
- Точечные оценки: как далеко от истины? Есть ли систематическая ошибка?
- Интервальные оценки: ширина, покрытие (coverage), разрешающая способность
- Компромисс: 100% покрытие → слишком широкий интервал

**Ключевые концепции и формулы:**
- Статистика T(X1,...,Xn) — функция только от данных
- Bias: E[T] - theta
- Несмещённость: E[T] = theta
- Доверительный интервал: P(theta in CI) >= 1 - alpha

**Для MLE собеседований:**
| Тема | Статус | Комментарий |
|------|--------|-------------|
| Bias vs Variance | 🟢 | Топ-вопрос: «Bias-variance tradeoff» |
| Доверительные интервалы | 🟢 | «Что такое 95% CI?» — частый вопрос |
| Статистика vs параметр | 🟡 | Понимание основ |
| Частотный vs байесовский подход | 🟢 | «Чем отличается CI от credible interval?» |

**Что вынести:** В частотном подходе параметр фиксирован, данные случайны. Оценка — случайная величина. Bias = систематическая ошибка. Доверительный интервал: при повторении эксперимента 95% интервалов покроют истинный параметр.

**Зависимости:** Лекции 5-6 (выборочные характеристики, ЦПТ).

---

### Лекция 9: Робастность. Bootstrap. Тяжёлые хвосты.

**Приоритет:** 🟢 Высокая

Тяжёлые хвосты: стандартное отклонение может быть обманчивым. Робастность (robustness): устойчивость к выбросам. Пороговая точка (breakdown point). Усечённое среднее, медиана. Bootstrap: параметрический и непараметрический. Ошибка bootstrap: неустранимая + устранимая. Практика: перформанс-метрики, бенчмарки.

**Основные темы:**
- Загрязнённые нормальные распределения: 5% «грязи» → стандартное отклонение x3-x11
- Робастность: устойчивость к выбросам
- Пороговая точка: минимальная доля выборки для «уничтожения» оценки
  - Выборочное среднее: 1/n (одного выброса достаточно!)
  - Медиана: ≈0.5 (супер-робастная)
  - k-усечённое среднее: k/n
- Методы робастного оценивания: усечение, винзоризация
- Bootstrap: непараметрический (из эмпирического распределения)
- Bootstrap: параметрический (из подогнанной модели)
- Ошибка bootstrap = неустранимая (аппроксимация) + устранимая (MC)
- Важность выбора модели для bootstrap

**Ключевые концепции и формулы:**
- Пороговая точка выборочного среднего = 1/n
- Пороговая точка медианы ≈ 0.5
- k-усечённое среднее: отбрасываем k наименьших и k наибольших
- Винзоризация: заменяем выбросы на ближайшие «нормальные» значения
- Bootstrap CI: генерируем B выборок → B оценок → квантили

**Для MLE собеседований:**
| Тема | Статус | Комментарий |
|------|--------|-------------|
| Bootstrap | 🟢 | «Как построить доверительный интервал без формул?» |
| Робастные оценки | 🟢 | Работа с выбросами — практический навык |
| Тяжёлые хвосты | 🟡 | Понимание для EDA |
| Медиана vs среднее | 🟢 | «Когда использовать медиану?» — практический вопрос |
| Квантили для перформанса (p99, p95) | 🟢 | SRE/SLA метрики, latency |

**Что вынести:** Среднее неробастно — один выброс может «убить» оценку. Медиана — самая робастная характеристика положения. Bootstrap — универсальный метод построения CI. Для перформанс-метрик используйте квантили (p95, p99), не среднее.

**Зависимости:** Лекции 5-6 (выборочные характеристики, MC).

### Лекция 10: p-value. Ошибки I и II рода. Множественные сравнения. Переобучение.

**Приоритет:** 🟢 **КРИТИЧЕСКАЯ**

Завершающая лекция. Детальный разбор p-value (определение через минимальный уровень значимости). Односторонние и двусторонние критерии. Ошибки I и II рода. Размер эффекта: разница между статистической значимостью и практической значительностью. Множественные сравнения (Бонферрони). p-hacking: «прогулка по саду». Переобучение в статистике.

**Основные темы:**
- p-value: минимальный уровень значимости, при котором отвергаем H0
- p-value = P(более экстремальное значение | H0)
- Односторонние vs двусторонние критерии
- Ошибка I рода (alpha): отвергли H0, когда она верна (ложная тревога)
- Ошибка II рода (beta): не отвергли H0, когда верна H1 (пропуск)
- Мощность критерия = 1 - beta
- Размер эффекта: разница, стандартизированная разница, относительная разница
- Значимость != значительность
- Множественные сравнения: проблема Бонферрони
- p-hacking: «прогулка по саду с ветвящимися тропинками»
- Переобучение: если долго подгонять модель — она описывает данные, а не распределение
- Решение: проверка на новых данных

**Ключевые концепции и формулы:**
- p-value = min{alpha : T(data) in C_alpha}
- Для правосторонней: p-value = P(T >= t_obs | H0)
- Для двусторонней: p-value = 2 * P(T >= |t_obs| | H0)
- Ошибка I рода: alpha = P(отвергнуть H0 | H0 верна)
- Ошибка II рода: beta = P(не отвергнуть H0 | H1 верна)
- Поправка Бонферрони: alpha_i = alpha / k (для k сравнений)
- Размер эффекта Коэна: d = (X_bar - mu0) / s

**Для MLE собеседований:**
| Тема | Статус | Комментарий |
|------|--------|-------------|
| p-value: определение и интерпретация | 🟢 | Топ-вопрос для A/B тестов: «Что такое p-value?» |
| Ошибки I и II рода | 🟢 | «Что такое false positive / false negative?» |
| Мощность критерия | 🟢 | «Как определить нужный размер выборки?» |
| Значимость vs значительность | 🟢 | «p-value маленький, а эффект мизерный — что делать?» |
| Множественные сравнения | 🟢 | A/B тесты с множеством метрик |
| p-hacking | 🟢 | Практическая грамотность |
| Переобучение | 🟢 | Фундаментальная проблема ML |

**Что вынести:** p-value — это НЕ вероятность того, что H0 верна. Это вероятность получить такие или более экстремальные данные при условии H0. Маленький p-value ≠ большой эффект. При множественных сравнениях нужна коррекция. Переобучение: проверяйте на новых данных.

**Зависимости:** Лекции 4, 8 (проверка гипотез, оценки).

## Итог

Курс Николаева — **лучшее введение в статистическое мышление** для ML-инженера на русском языке. Он не даёт формальных доказательств, но даёт **интуицию**: почему мы используем MLE, что такое p-value на самом деле, почему bootstrap работает, в чём разница между байесовским и частотным подходом. Эта интуиция критична для собеседований, где нужно не воспроизвести формулу, а объяснить *почему*.

**Уникальная ценность курса:**
- Сквозной пример (игровой автомат) делает абстрактные концепции осязаемыми
- Явное сравнение частотного и байесовского подходов
- Bootstrap как центральная техника (а не приложение)
- Практические примеры: перформанс-метрики, A/B тесты, кризис воспроизводимости
- Разница между значимостью и значительностью — то, что часто не понимают на собесах
