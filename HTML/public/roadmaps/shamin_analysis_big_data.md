### Л2: Линейная регрессия
**Приоритет:** 🟣 Обязательно

**Содержание:** Парная и множественная регрессия, МНК, оптимизация функции двух переменных, анализ остатков, гауссовский белый шум, коэффициент детерминации R², нормировка данных.

**Зачем NLP/LLM-инженеру:**

- R² и интерпретация коэффициентов — пригодится при feature importance
- Множественная регрессия в матричной форме — фундамент для понимания линейных слоёв нейросетей

**На что обратить внимание:**
- [30:00] Оптимизация через частные производные — это и есть gradient descent
- [55:00] Гауссовский белый шум — идеал остатков, связь с предположениями MLE
- [01:00:00] R² — запомнить, что «хороший R²» зависит от домена

### Л5: Классификация и кластеризация
**Приоритет:** 🟣 Обязательно

**Содержание:** Переход от временных рядов к векторам объектов, метрика Евклида, нормировка Min-Max, алгоритм Кохонена, обучение с/без учителя, PCA, автокодировщики (матричная форма), белый vs чёрный ящик.

**Зачем NLP/LLM-инженеру:**
- Векторное представление объектов — это и есть embeddings
- Нормировка данных → LayerNorm, BatchNorm в трансформерах
- PCA — понимание того, как снижать размерность (используется в визуализации эмбеддингов)
- Автокодировщики в матричной форме — A сжимает, B восстанавливает, A×B ≈ I — прямой путь к VAE и seq2seq
- Supervised vs Unsupervised — базовое разделение, которое спрашивают на собесах

**На что обратить внимание:**
- [00:00] Каждый объект = вектор в R^n — это мышление, на котором строятся эмбеддинги
- Блок про PCA — связать с SVD из своих знаний
- Блок про автокодировщики — encoder-decoder, бутылочное горлышко

## Фаза 2 — Ключевые алгоритмы ML (Core)

> Ядро курса. Прямо применимо к ML-инженерии.

### Л9: Решающие деревья
**Приоритет:** 🟢 Ключевая

**Содержание:** Деревья решений vs нейросети vs базы данных, интерпретируемость, теория информации Шеннона, энтропия, информационный выигрыш (Information Gain), критерий Джини, построение дерева, pruning.

**Зачем NLP/LLM-инженеру:**
- **Энтропия Шеннона H = −Σ pᵢ log pᵢ** — это то, что стоит внутри cross-entropy loss при обучении GPT/BERT
- Information Gain — формализация «сколько информации даёт разбиение» — та же логика в attention (какие токены информативны)
- Интерпретируемость vs точность — центральный tradeoff в ML, особенно актуален для LLM (explainability)
- Деревья — база для бустинга (Л10), а gradient boosted trees до сих пор бьют нейросети на табличных данных

**На что обратить внимание:**
- [10:00] Энтропия — формула, интуиция, связать с cross-entropy из своего бэкграунда
- Принцип построения дерева — жадный алгоритм, выбор лучшего признака
- Белый ящик — почему бизнес любит деревья

---

### Л10: Ансамбли моделей (Бустинг и Бэггинг)
**Приоритет:** 🟢 Ключевая

**Содержание:** Философия ансамблей (много слабых > одного сильного), бустинг (AdaBoost — перевзвешивание ошибок), бэггинг (bootstrap + параллельные модели), Random Forest, потеря интерпретируемости, философия «грубой вычислительной силы».

**Зачем NLP/LLM-инженеру:**
- Бустинг — последовательное исправление ошибок, та же идея в residual connections (ResNet, Transformer)
- Бэггинг — параллельные независимые модели → Mixture of Experts (MoE) в современных LLM
- Random Forest — до сих пор один из лучших методов для табличных данных, часто используется как baseline
- Понимание «слабый классификатор + композиция = сильный» — фундаментальная идея ML

**На что обратить внимание:**
- [05:00] Пример Гальтона — интуиция ансамблей
- [10:00] Математика бустинга — веса объектов, итеративное обучение
- [25:00] Бэггинг и bootstrap — связать с bootstrap из статистики (уже знаешь)
- [35:00] Random Forest — комитет голосования

---

### Л8: Сэмплирование данных
**Приоритет:** 🟢 Ключевая

**Содержание:** Генеральная совокупность, репрезентативность, детерминированный vs вероятностный сэмплинг, закон разнообразия Эшби, стратифицированный и кластеризованный сэмплинг, train/test split, snowball sampling, удобный сэмплинг, анонс решающих деревьев.

**Зачем NLP/LLM-инженеру:**
- Train/test split — делаешь каждый день, но важно понимать почему именно так
- Стратификация — критична при несбалансированных классах (типичная проблема в NLP: NER, sentiment, classification)
- Data leakage — понимание того, почему тестовая выборка должна быть изолирована
- Нестационарность временного сэмплинга — релевантно для fine-tuning LLM на свежих данных

**На что обратить внимание:**
- [40:00] Train/test split для нейросетей — правила и подводные камни
- [25:00] Стратификация — связать с k-fold (уже знаешь обзорно)
- [45:00] Snowball sampling — интересно для понимания графовых структур (knowledge graphs в NLP)

## Фаза 3 — Временные ряды (обзорно)

> Полезный кругозор. Можно смотреть на 1.5× скорости, останавливаясь на ключевых блоках.

### Л3: Временные ряды (теория)
**Приоритет:** 🟠 Обзорно

**Содержание:** Специфика временных рядов, тренд/сезонность/цикличность, декомпозиция ряда, скользящее среднее, экспоненциальное сглаживание, автокорреляция.

**Зачем NLP/LLM-инженеру:**
- Авторегрессия — GPT это по сути авторегрессионная модель (предсказывает следующий токен по предыдущим)
- Концепция «текущее значение зависит от предыдущих контекстных значений» = контекстное окно LLM
- Стационарность — понимание того, почему данные из прошлого могут не работать для будущего (distribution shift)

**Смотреть ускоренно, остановиться на:**
- [00:00] Введение — аналогия с авторегрессионными моделями
- Блок про декомпозицию — отделение сигнала от шума (пригодится для data cleaning)

### Л4: ARIMA + Python
**Приоритет:** 🟠 Обзорно

**Содержание:** Вероятностное пространство (Ω, σ-алгебра, мера P), случайные величины, случайные процессы, траектории, модели AR, MA, ARMA, ARIMA, дифференцирование рядов, практика в Python (statsmodels).

**Зачем NLP/LLM-инженеру:**
- Вероятностное пространство — повторение и углубление того, что уже знаешь (Колмогоров, σ-алгебра)
- Модель AR(p) — прямая аналогия: y_t = a₁·y_{t-1} + ... + aₚ·y_{t-p} — это авторегрессия, GPT делает то же самое через attention
- ARIMA в Python — пример полного ML-пайплайна: данные → модель → тюнинг → прогноз

**Смотреть ускоренно, остановиться на:**
- [05:00–20:00] Вероятностное пространство — быстро пролистать, ты это уже знаешь
- [30:00] Модель AR — понять связь с авторегрессией в LLM
- [50:00] Python практика — полезно увидеть пайплайн

## Фаза 4 — Визуализация и SOM (факультативно)

> Расширение кругозора. Менее критично для NLP/LLM. Пропуск без потерь при нехватке времени.

### Л6: Карты Кохонена (SOM) — теория
**Приоритет:** ⚪ Факультативно

**Содержание:** Самоорганизующиеся карты, проекция многомерных данных на 2D, топология нейронов, BMU, функция соседства, экспоненциальное затухание, мониторинг динамических систем, фазовые траектории.

**Зачем NLP/LLM-инженеру:**
- Общее понимание unsupervised learning и снижения размерности
- Идея нейронов-победителей — отголоски в winner-take-all attention
- В целом — исторический контекст, SOM в NLP практически не используется

**Зачем NLP/LLM-инженеру:**
