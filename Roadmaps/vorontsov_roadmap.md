МНК (метод наименьших квадратов), нормальное уравнение, Ridge/Lasso, мультиколлинеарность, SVD, PCA (метод главных компонент), снижение размерности.

| Временной блок | Содержание | Статус | Для собесов |
|----------------|-----------|--------|-------------|
| 0:00–0:15 | МНК: нормальное уравнение (F^T F)α = F^T y | 🟢 Смотреть | «Как аналитически решить линейную регрессию?» |
| 0:15–0:30 | ⭐ **Мультиколлинеарность, почему матрица вырождена** | 🟢 Ключевое | Зачем нужна регуляризация |
| 0:30–0:45 | ⭐ **Ridge (L2) vs Lasso (L1): геометрическая интерпретация** | 🟢 Ключевое | L1 = ромб → разреженность, L2 = шар → сжатие |
| 0:45–1:05 | ⭐ **SVD: разложение A = UΣV^T, связь с рангом, low-rank аппроксимация** | 🟢 Ключевое | SVD — фундамент для PCA и LoRA |
| 1:05–1:25 | ⭐ **PCA: собственные значения, explained variance, scree plot** | 🟢 Ключевое | «Как связаны SVD и PCA?» — стандартный вопрос |

**⭐ Главные инсайты:**
- Ridge добавляет λI к F^T F → делает матрицу обратимой (собственные значения += λ)
- Lasso обнуляет мелкие коэффициенты → автоматический отбор признаков
- SVD: сингулярные значения σ₁ ≥ σ₂ ≥ ... показывают «важность» каждого направления
- PCA = SVD матрицы данных → собственные вектора ковариационной матрицы
- Low-rank аппроксимация через SVD → основа LoRA для LLM

**Связь с собесами:** «Связь SVD и PCA?» — в плане обучения (learn_state) это в списке тем Блока A. Воронцов даёт лучшее объяснение.

**Связь с собесами:** «Объясните разницу precision vs recall, когда что важнее?» — спрашивают в КАЖДОЙ российской компании.

| 1:20–1:35 | Сравнение методов, практические рекомендации | 🟢 Смотреть | Когда RF vs boosting |

**⭐ Главные инсайты:**
- Bagging снижает дисперсию: Var(среднего) = Var(одного)/n (при независимости)
- RF добавляет рандомизацию признаков → деревья более «разные» → лучше ансамбль
- AdaBoost: на каждом шаге увеличивает веса «трудных» объектов → фокусирует обучение
- AdaBoost минимизирует экспоненциальный loss = верхнюю оценку числа ошибок

**Связь с собесами:** «Bagging vs Boosting — в чём разница?» — один из самых частых вопросов.

### Лекция 11: Gradient Boosting, XGBoost, CatBoost ⭐

**Длительность:** ~90 мин | **Приоритет:** 🟢 **КРИТИЧЕСКАЯ**

Gradient Boosting: идея функционального градиентного спуска, XGBoost: регуляризация деревьев + split-finding, работа с категориальными признаками, стохастический GB, bias-variance декомпозиция.

| Временной блок | Содержание | Статус | Для собесов |
|----------------|-----------|--------|-------------|
| 0:00–0:15 | ⭐ **Идея GB: каждое дерево приближает антиградиент loss** | 🟢 Ключевое | «Как работает Gradient Boosting?» — топ-вопрос |
| 0:15–0:30 | ⭐ **GB для MSE: каждое дерево учится на остатках** | 🟢 Ключевое | Самая понятная интуиция |
| 0:30–0:50 | ⭐ **XGBoost: регуляризация + approx split + аппаратные оптимизации** | 🟢 Ключевое | XGBoost/LightGBM/CatBoost — на собесах в РФ |
| 0:50–1:05 | ⭐ **Bias-variance tradeoff в контексте бустинга** | 🟢 Ключевое | Boosting снижает bias, bagging снижает variance |
| 1:05–1:15 | Категориальные признаки, target encoding, CatBoost-идея | 🟢 Смотреть | CatBoost — специфика рынка РФ |
| 1:15–1:30 | Stochastic GB, regularization tricks, early stopping | 🟢 Смотреть | Практические советы |

**⭐ Главные инсайты:**
- GB = функциональный градиентный спуск в пространстве алгоритмов
- Каждый новый базовый алгоритм bₜ(x) приближает −∇L на обучающих объектах
- Для MSE: антиградиент = остатки (y − ŷ) → дерево учится на ошибках предыдущих
- Boosting снижает bias (компенсирует «недообучение»), но может повысить variance → нужен shrinkage (learning rate η < 1)
- XGBoost: второй порядок Тейлора → более точная аппроксимация gain при split

**Связь с собесами:** GB + XGBoost/CatBoost — **главный алгоритм для табличных данных в индустрии**. Спрашивают в Яндексе, Сбере, Т-Банке, Ozon. Без понимания GB нельзя пройти MLE-собес в РФ.

### Лекция 13: Байесовская классификация, Naive Bayes

**Длительность:** ~95 мин | **Приоритет:** 🟢 Высокая

Оптимальный байесовский классификатор, генеративный vs дискриминативный подход, Naive Bayes (для числовых и текстовых признаков), гауссовский байесовский классификатор, LDA/QDA.

| Временной блок | Содержание | Статус | Для собесов |
|----------------|-----------|--------|-------------|
| 0:00–0:15 | ⭐ **Оптимальный байесовский классификатор: формула, интерпретация** | 🟢 Ключевое | P(y|x) = P(x|y)·P(y) / P(x) → argmax |
| 0:15–0:25 | Генеративный vs дискриминативный подход | 🟢 Смотреть | Важное концептуальное различие |
| 0:25–0:45 | ⭐ **Naive Bayes: предположение независимости, текстовая классификация** | 🟢 Ключевое | NB для текстов = мешок слов + мультиномиальная модель |
| 0:45–1:05 | ⭐ **Гауссовский классификатор: LDA, QDA, связь с логрег** | 🟢 Ключевое | LDA = линейный, QDA = квадратичный байесовский |
| 1:05–1:20 | Метрический байесовский классификатор, смеси для классов | 🟡 Полезно | Связь density estimation → classification |
| 1:20–1:35 | Metric learning как следствие | 🟡 Опционально | Продвинутая тема |

**⭐ Главные инсайты:**
- Оптимальный байесовский классификатор: выбирай класс с максимальной P(y|x) — минимизирует ожидаемые потери
- Naive Bayes для текстов: P(word|class) считается по частотам → «мешок слов»
- LDA = Naive Bayes с гауссовскими признаками + одинаковая ковариация → линейная граница
- QDA = разные ковариации → квадратичная граница

| 0:15–0:30 | ⭐ **K-Means++: инициализация, elbow method, silhouette** | 🟢 Ключевое | Практические улучшения |

**⭐ Главные инсайты:**

- Иерархическая: не нужно заранее знать k, но O(n²) по памяти
