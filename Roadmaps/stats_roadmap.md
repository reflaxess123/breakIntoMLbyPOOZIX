**⭐ Главные инсайты:**
- **LLN:** X̄_n →ᵖ μ при n→∞ — среднее выборки сходится к мат. ожиданию
- **CLT:** √n(X̄_n − μ)/σ →ᵈ N(0,1) — среднее выборки ~ нормально при большом n, **независимо от исходного распределения**
- CLT — **почему работает всё**: доверительные интервалы, t-тесты, A/B-тесты
- Проклятие размерности: в d-мерном пространстве объём шара → 0 при d→∞ → KNN перестаёт работать

**Связь с MLE-собесами:** «Сформулируйте ЦПТ» — стандартный вопрос. «Почему нормальное распределение так часто встречается?» — ответ: ЦПТ.

**⭐ Главные инсайты:**

- Ширина CI ∝ 1/√n → для уменьшения ширины вдвое нужно в 4 раза больше данных
- Дельта-метод: если θ̂ ~ N(θ, σ²/n), то g(θ̂) ~ N(g(θ), [g'(θ)]²·σ²/n)

| 07-05–06 | ⭐ **Несмещённость и состоятельность: формулы, примеры** | 🟢 Ключевое |
| 07-07 | ⭐ **Состоятельность ≠ несмещённость (контрпримеры)** | 🟢 Ключевое |
| 07-08–09 | ⭐ **Bias-Variance Tradeoff, регуляризация** | 🟢 Ключевое |
| 07-10–11 | Эффективность, неравенство Крамера-Рао | 🟡 Для эрудиции |

**⭐ Главные инсайты:**
- **Несмещённость:** E[θ̂] = θ (в среднем не врёт)
- **Состоятельность:** θ̂_n →ᵖ θ (с ростом выборки приближается к правде)
- **Эффективность:** Var(θ̂) достигает нижней границы Крамера-Рао
- ⭐ **Bias-Variance Tradeoff:** MSE(θ̂) = Var(θ̂) + Bias²(θ̂) — **это ПРЯМО вопрос на MLE-собесах**
- Регуляризация добавляет bias, но уменьшает variance → общая ошибка может уменьшиться

**Связь с MLE-собесами:** Bias-variance tradeoff = один из «5 обязательных выводов». Связь с L2 регуляризацией = Ridge = байесовский prior.

### Неделя 9: Проверка гипотез: p-value, ошибки I/II рода ⭐

**Приоритет:** 🟢 **КРИТИЧЕСКАЯ**

| Секция | Содержание | Статус |
|--------|-----------|--------|
| 09-01 | ⭐ **Проверка гипотез: H₀ vs H₁, логика** | 🟢 Ключевое |
| 09-02 | ⭐ **Процедура: статистика → распределение → p-value → решение** | 🟢 Ключевое |

| 09-04–06 | ⭐ **Ошибки I рода (α) и II рода (β), мощность (1−β)** | 🟢 Ключевое |
| 09-07 | ⭐ **Сколько наблюдений нужно (sample size / power analysis)** | 🟢 Ключевое |
| 09-08 | Параметрические критерии (z-тест, t-тест) | 🟢 Смотреть |

| 09-11 | Гипотезы о средних (одновыборочный, двухвыборочный t-тест) | 🟢 Смотреть |
| 09-12–13 | Гипотезы о дисперсиях (F-тест) | 🟡 Для эрудиции |
| 09-14 | ⭐ **Критерии для зависимых выборок (парный t-тест)** | 🟢 Смотреть |

**⭐ Главные инсайты:**
- **p-value:** вероятность получить такую же или более экстремальную статистику при H₀. **НЕ** вероятность H₀!
- **Ошибка I рода (α):** отвергнуть H₀, когда она верна (false positive)
- **Ошибка II рода (β):** не отвергнуть H₀, когда верна H₁ (false negative)
- **Мощность = 1−β:** вероятность обнаружить эффект, если он есть
- **Sample size:** n ∝ (z_α + z_β)² · σ² / δ² — чем меньше эффект δ, тем больше данных нужно

**Связь с MLE-собесами:** «Что такое p-value?» — **самый частый статистический вопрос** на DS/MLE-собесах. Ошибки I/II рода → precision/recall аналогия.

| 11-04–05 | ⭐ **Множественное тестирование: Бонферрони, FDR, BH-процедура** | 🟢 Ключевое |
| 11-06–07 | ⭐ **Расчёт необходимого числа наблюдений (sample size)** | 🟢 Ключевое |
| 11-08 | ⭐ **Метрики для A/B: конверсия, ARPU, retention** | 🟢 Ключевое |
| 11-09 | Практический кейс | 🟡 Для закрепления |

**⭐ Главные инсайты:**
- A/B-тест = двухвыборочный тест гипотез в продуктовом контексте
- **Peeking problem:** если смотришь на результаты до окончания → inflated false positive rate → нужен sequential testing
- **Множественное тестирование:** если k гипотез при α=0.05, вероятность хотя бы 1 ложного срабатывания ≈ 1−(1−α)^k
- **Bonferroni:** α_adj = α/k (консервативно)
- **BH (Benjamini-Hochberg):** контролирует FDR (false discovery rate) — менее консервативно
- **Sample size:** зависит от α, β, MDE (минимальный детектируемый эффект), дисперсии метрики

**Связь с MLE-собесами:** A/B-тесты спрашивают в **каждой** российской продуктовой компании: Яндекс, Сбер, Ozon, VK, T-Bank. Эта неделя — подготовка к целой секции собесования.

| 12-09–10 | ⭐ **Тест отношения правдоподобий (likelihood ratio test)** | 🟢 Ключевое |
| 12-11–12 | Дельта-метод (повторение) | 🟡 Полезно |
| 12-13–14 | MLE в Python, пуассоновская регрессия | 🟡 Практика |

**⭐ Главные инсайты:**
- **MLE:** θ̂ = argmax_θ Π P(xᵢ|θ) = argmax_θ Σ log P(xᵢ|θ)
- Логарифм превращает произведение в сумму → удобнее оптимизировать
- MLE-оценки **состоятельны, асимптотически нормальны, асимптотически эффективны**
- **Информация Фишера:** I(θ) = −E[∂²logL/∂θ²] — «кривизна» логарифма правдоподобия
- Var(θ̂_MLE) → 1/(n·I(θ)) — связь с неравенством Крамера-Рао

**Связь с MLE-собесами:** «Выведите cross-entropy через MLE» — один из «5 обязательных выводов». MLE = основа обучения почти всех ML-моделей.

### Неделя 13: Линейная регрессия — MLE↔Loss Functions, Bias-Variance ⭐⭐

**Приоритет:** 🟢 **КРИТИЧЕСКАЯ — САМАЯ ВАЖНАЯ НЕДЕЛЯ КУРСА**

| Секция | Содержание | Статус |
|--------|-----------|--------|
| 13-01 | План недели | 🟡 Быстро |
| 13-02 | ⭐ **Линейная регрессия через MLE** | 🟢 Ключевое |
| 13-03 | ⭐ **MSE = MLE при гауссовском шуме** | 🟢 Ключевое |
| 13-04 | ⭐ **MAE = MLE при лапласовском шуме** | 🟢 Ключевое |
| 13-05 | ⭐ **Logloss = MLE при Бернулли (логистическая регрессия)** | 🟢 Ключевое |
| 13-06 | Функция потерь для Пуассона | 🟡 Расширение |
| 13-07 | ⭐ **MSE → модель предсказывает среднее; MAE → медиану** | 🟢 Ключевое |
| 13-08 | Logloss → модель предсказывает вероятность | 🟢 Ключевое |
| 13-09 | Когда важна интерпретация | 🟢 Смотреть |
| 13-10 | Статистический подход к регрессии | 🟢 Смотреть |
| 13-11 | ⭐ **Проверка гипотез в регрессии (значимость коэффициентов)** | 🟢 Ключевое |
| 13-12 | Проблемы регрессии: мультиколлинеарность, гетероскедастичность | 🟢 Смотреть |
| 13-15 | ⭐ **Bias-Variance Decomposition: E[(ŷ−y)²] = Bias² + Variance + Noise** | 🟢 Ключевое |
| 13-16–17 | Доска: вывод для парной регрессии | 🟡 Для глубины |

**⭐ Главные инсайты:**
- **MSE = MLE при y = Xβ + ε, ε~N(0,σ²)** → минимизация MSE = максимизация likelihood при нормальном шуме
- **MAE = MLE при ε~Laplace** → MSE vs MAE = гауссовский vs лапласовский шум
- **Logloss = MLE при y~Bernoulli(σ(Xβ))** → cross-entropy loss = negative log-likelihood
- **MSE → модель предсказывает E[Y|X]; MAE → модель предсказывает медиану Y|X**
- **Bias-Variance:** MSE = Bias² + Variance + irreducible noise → регуляризация увеличивает bias, уменьшает variance

**Связь с MLE-собесами:** Эта неделя покрывает 3 из «5 обязательных выводов»: MLE↔MSE, MLE↔cross-entropy, bias-variance tradeoff. Это **THE лекция** для понимания, почему в ML используются именно эти loss-функции.

### Неделя 16: Байесовский подход

**Приоритет:** 🟢 Высокая

| Секция | Содержание | Статус |
|--------|-----------|--------|
| 16-01 | ⭐ **Что такое байесовский подход: prior → data → posterior** | 🟢 Ключевое |
| 16-02–03 | ⭐ **Формула Байеса: P(θ|D) ∝ P(D|θ)·P(θ)** | 🟢 Ключевое |
| 16-04 | Частотная vs байесовская статистика | 🟢 Смотреть |
| 16-05–07 | ⭐ **Байесовский вывод и прогнозирование на примерах** | 🟢 Ключевое |
| 16-08 | ⭐ **Сопряжённые распределения** | 🟢 Смотреть |
| 16-09 | ⭐ **MCMC: идея Metropolis-Hastings** | 🟡 Полезно |
| 16-10–11 | PyMC3: байесовские модели на практике | 🟡 Практика |

**⭐ Главные инсайты:**
- **Байесовский подход:** параметр θ — случайная величина с распределением (prior), данные обновляют это распределение (posterior)
- **P(θ|D) = P(D|θ)·P(θ) / P(D)** — posterior ∝ likelihood × prior
- **Сопряжённые:** если prior и posterior из одного семейства → аналитическое решение (Beta-Bernoulli, Normal-Normal)
- **MCMC:** когда P(D) не считается аналитически → сэмплируем из posterior

**Связь с MLE-собесами:** Регуляризация = байесовский prior (L2 = N(0,σ²), L1 = Laplace). RLHF использует байесовский фреймворк. Байесовская оптимизация — для подбора гиперпараметров.
