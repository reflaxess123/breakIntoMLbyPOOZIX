### Лекция 3: Байесовское обучение, Naive Bayes ⭐

**Длительность:** ~80 мин | **Релевантность для MLE:** 🟢 **ВЫСОКАЯ** | **СМОТРЕТЬ ЦЕЛИКОМ**

Центральная лекция курса для MLE. Полный рабочий пайплайн байесовского классификатора от теории до числового примера.

| Временной блок | Содержание | Статус | Комментарий |
|----------------|-----------|--------|-------------|
| 0:00–0:10 | Введение: «Байесовское обучение — это и есть настоящее ML» | 🟢 Контекст | Мотивация, почему это работает |
| 0:10–0:25 | ⭐ **Условная вероятность → формула Байеса с разбиением** | 🟢 Ключевое | P(Aₖ|D) = P(D|Aₖ)·P(Aₖ) / Σ P(D|Aᵢ)·P(Aᵢ). MAP-гипотеза. Вывод на доске |
| 0:25–0:40 | ⭐ **Парадокс медицинского теста** | 🟢 Ключевое | 1% больных, тест 95% точный → положительный результат = только 16% вероятность болезни. Классическая задача на собесах! |
| 0:40–0:55 | ⭐ **Naive Bayes классификатор: предположение независимости** | 🟢 Ключевое | P(D|Aₖ) = ∏ P(Dᵢ|Aₖ). Почему «наивное» предположение работает лучше точной модели |
| 0:55–1:05 | ⭐ **Числовой пример: 350 человек, 4 признака** | 🟢 Ключевое | Полный расчёт: таблица частот → апостериорные вероятности → результат 65% женщина |
| 1:05–1:10 | Сглаживание Лапласа (проблема нулевых вероятностей) | 🟢 Важно | Практический трюк: +1 ко всем счётчикам |
| 1:10–1:20 | Введение матожидания (ступенчатые функции) | 🟡 Контекст | Мостик к следующей лекции |

**Ключевые формулы:**
- Байес: P(Aₖ|D) = P(D|Aₖ)·P(Aₖ) / Σᵢ P(D|Aᵢ)·P(Aᵢ)
- MAP: A*ₖ = argmax P(Aₖ|D)
- Naive Bayes: P(D|Aₖ) = P(D₁|Aₖ)·P(D₂|Aₖ)·...·P(Dₙ|Aₖ)

**⭐ Главный инсайт лекции:** Naive Bayes работает лучше точной модели потому что: (1) точные зависимости требуют огромных данных, (2) на ограниченных данных «наивная» модель устойчивее, (3) «ищем кошелёк под фонарём, где светлее» — упрощение модели компенсирует ошибку предположения. Это фундаментальный принцип ML: **простая неправильная модель часто бьёт сложную правильную**.

**Связь с собесами:** Теорема Байеса + Naive Bayes + парадокс медицинского теста — стандартные вопросы в Яндексе, Т-Банке, Сбере (документ «Какую глубину математики требуют»).

### Лекция 8: Нейронные сети, Backpropagation ⭐

**Длительность:** ~90 мин | **Релевантность для MLE:** 🟢 **ВЫСОКАЯ** | **СМОТРЕТЬ ЦЕЛИКОМ**

Ядро курса для MLE. Перцептрон, двухслойная сеть, сигмоида, backpropagation «готовый к программированию».

| Временной блок | Содержание | Статус | Комментарий |
|----------------|-----------|--------|-------------|
| 0:00–0:15 | ⭐ **Постановка задачи: интерполяция в пространстве 2ᴺ** | 🟢 Ключевое | ML = интерполяция. Train/test = 80/20. Чёткая формализация |
| 0:15–0:35 | ⭐ **Перцептрон: алгоритм обучения, XOR-проблема, «зима AI»** | 🟢 Ключевое | Исторический контекст: почему один слой недостаточен |
| 0:35–0:55 | ⭐ **Двухслойная сеть, сигмоида: σ(t) = 1/(1+e⁻ᵃᵗ)** | 🟢 Ключевое | σ'(t) = σ(t)·(1−σ(t)) — производная через саму себя. Архитектура: вход → скрытый → выход |
| 0:55–1:20 | ⭐ **Backpropagation — полный пошаговый вывод** | 🟢 Ключевое | «Готово к программированию»: 7 шагов от инициализации до обновления весов. Самый ценный фрагмент курса |
| 1:20–1:30 | ⭐ **Переобучение, «чёрный ящик», размер данных** | 🟢 Ключевое | Большой H → сеть «запоминает» вместо обобщения. Пример: торговая система, работавшая на истории и провалившаяся на реальных данных |

**Ключевые формулы:**
- Перцептрон: y = sign(w·x), обновление: w ← w + λ·d·x (если ошибка)
- Сигмоида: σ(t) = 1/(1+e⁻ᵃᵗ), σ'(t) = σ(t)·(1−σ(t))
- Backprop (output): w₂[h,m] −= λ·e[m]·σ'(out₂[m])·z[h]
- Backprop (hidden): w₁[n,h] −= λ·e₂[h]·σ'(out₁[h])·x[n]

**⭐ Главный инсайт лекции:** Шамин даёт backprop не как абстрактную теорию, а как конкретный алгоритм из 7 шагов, который можно сесть и закодить. Плюс честный разговор о проблемах: «Веса w₁ и w₂ ничего не говорят о ТОМ, КАК сеть принимает решения — это чёрный ящик».

**Связь с собесами:** Backpropagation — один из «пяти обязательных выводов» для MLE (документ «Какую глубину математики требуют»). σ'(z) = σ(z)(1−σ(z)) — стандартный вопрос Stanford CS230.

| 0:00–0:10 | ⭐ **Постановка задачи регрессии, числовой пример** | 🟢 Ключевое | 5 точек {1,2,5,7,10} → построить дерево |
| 0:10–0:25 | ⭐ **Три меры беспорядка: дисперсия, энтропия Шеннона, RSS** | 🟢 Ключевое | Когда какой критерий: RSS для регрессии, энтропия для классификации |
| 0:25–0:40 | ⭐ **Ручное построение дерева с вычислением RSS** | 🟢 Ключевое | Предикат «x ≤ 2»: RSS = 16. Предикат «x ≤ 5»: RSS = 13.17 — лучше. Рекурсия вглубь |
| 0:40–0:50 | ⭐ **Random Forest: алгоритм, зачем, как** | 🟢 Ключевое | Бутстрэп → M деревьев → усреднение. «Белый ящик» одного дерева vs «чёрный ящик» леса |
| 0:50–1:00 | Пример: популяция ктенофоры в Чёрном море | 🟡 Интересно | Реальный кейс: дерево показало, что температура важнее хищника |

**Ключевые формулы:**
- RSS = Σ(yᵢ − ȳ)² (внутри каждого листа)
- Энтропия Шеннона: H = −Σ pᵢ log₂ pᵢ
- Random Forest: ŷ = (1/M) Σ Tₘ(x)

**⭐ Главный инсайт лекции:** Одно дерево — **белый ящик** (можно объяснить каждое решение), лес — **чёрный ящик** (точнее, но непрозрачен). Пример с биологами: учёные приняли результат дерева именно потому что оно показало логику рассуждения.

**Связь с собесами:** Decision trees + Random Forest — стандартные вопросы. Gradient Boosting (XGBoost, CatBoost) строится поверх деревьев, поэтому без понимания деревьев — никуда. На собесах спрашивают: «В чём разница bagging vs boosting?», «Как дерево выбирает split?», «Gini vs Entropy?»
