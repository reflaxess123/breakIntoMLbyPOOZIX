# Роадмап: К.В. Воронцов «Прикладные модели машинного обучения» (2-й семестр)

> **Цель:** Извлечь максимум для подготовки к MLE собеседованиям (NLP/LLM фокус).
> **Метод:** Полный анализ текстов 10 лекций (~750K символов).
> **Общая длительность курса:** ~14.5 часов.
> **Рекомендуемое время просмотра:** ~10 часов (70% курса).

---

## Контекст: зачем нужен 2-й семестр

1-й семестр Воронцова — **классический ML** (линейные модели, деревья, бустинг, метрики).
2-й семестр — **глубокое обучение + прикладные задачи**: CNN, RNN/LSTM, Transformer/Attention, Word2Vec, Autoencoders, VAE, GAN, рекомендательные системы, ранжирование, topic modeling, временные ряды, online learning.

Для целевой позиции **NLP/LLM ML Engineer** 2-й семестр **критически важнее** 1-го: здесь Transformer, Attention, эмбеддинги текстов — всё то, что составляет ядро NLP-стека.

### Лекция 1: CNN (подробно), ResNet, BatchNorm, RNN/LSTM/GRU ⭐

**Длительность:** ~95 мин | **Приоритет:** 🟢 **КРИТИЧЕСКАЯ**

Полноценное введение в Deep Learning: свёрточные сети (архитектура, pooling, stride), ResNet (skip connections), BatchNorm, далее — рекуррентные сети (vanilla RNN, vanishing gradients, LSTM, GRU).

| Временной блок | Содержание | Статус | Для собесов |
|----------------|-----------|--------|-------------|
| 0:00–0:15 | Обзор семестра, архитектуры нейросетей | 🟡 Быстро | Контекст |
| 0:15–0:35 | ⭐ **CNN: свёртки, padding, stride, pooling, архитектуры** | 🟢 Ключевое | «Как работает свёрточный слой?» |
| 0:35–0:50 | ⭐ **ResNet: skip connections, почему работает** | 🟢 Ключевое | Residual connections — в каждом Transformer |
| 0:50–1:05 | ⭐ **BatchNorm: зачем, как, где ставить** | 🟢 Ключевое | BatchNorm vs LayerNorm — вопрос на DL-собесах |
| 1:05–1:25 | ⭐ **RNN → vanishing gradients → LSTM → GRU** | 🟢 Ключевое | «Почему LSTM решает проблему vanishing gradients?» |
| 1:25–1:35 | GRU: упрощённый LSTM, bidirectional RNN | 🟢 Смотреть | GRU vs LSTM — стандартный вопрос |

**⭐ Главные инсайты:**
- CNN: каждый фильтр «ищет» определённый паттерн; pooling даёт инвариантность к сдвигу
- ResNet: f(x) + x — сети учат «остаток» (residual), а не полное преобразование → можно строить 100+ слоёв
- BatchNorm нормализует активации внутри mini-batch → стабилизирует обучение, позволяет больший learning rate
- LSTM: три гейта (forget, input, output) + cell state → информация может «течь» через много шагов без затухания
- GRU: два гейта (reset, update) — проще LSTM, часто работает не хуже

**Связь с собесами:** Эта лекция — Фаза 3 роадмапа (Deep Learning). CNN, LSTM, ResNet, BatchNorm — всё спрашивают. Residual connections и LayerNorm — ключевые компоненты Transformer.

### Лекция 2: Autoencoders, VAE, GAN, Transfer Learning

**Длительность:** ~100 мин | **Приоритет:** 🟢 Высокая

Автокодировщики (AE), разреженные AE, denoising AE, вариационные AE (VAE), генеративно-состязательные сети (GAN), transfer learning.

| Временной блок | Содержание | Статус | Для собесов |
|----------------|-----------|--------|-------------|
| 0:00–0:15 | Кластеризация нейросетями, deep clustering | 🟡 Быстро | Контекст |
| 0:15–0:35 | ⭐ **Autoencoders: encoder + decoder, bottleneck, связь с PCA** | 🟢 Ключевое | AE с линейной активацией = PCA |
| 0:35–0:50 | ⭐ **Sparse AE, denoising AE, KL-дивергенция для регуляризации** | 🟢 Смотреть | KL-дивергенция — нужна для VAE и LLM |
| 0:50–1:10 | ⭐ **VAE: reparametrization trick, ELBO, генерация** | 🟢 Ключевое | VAE — стандартный вопрос на DL-собесах |
| 1:10–1:25 | ⭐ **Transfer Learning: предобученные сети, fine-tuning слоёв** | 🟢 Ключевое | Fine-tuning = основа работы с LLM |
| 1:25–1:40 | ⭐ **GAN: генератор vs дискриминатор, min-max игра** | 🟢 Ключевое | «Как работает GAN?» |

**⭐ Главные инсайты:**
- AE учится сжимать данные в bottleneck → если bottleneck мал, AE выучивает самые важные признаки
- VAE: z ~ N(μ, σ²), reparametrization trick: z = μ + σ·ε (ε ~ N(0,1)) → можно backprop через sampling
- ELBO = reconstruction loss + KL-дивергенция (регуляризация латентного пространства)
- Transfer Learning: заморозить нижние слои предобученной сети, дообучить верхние → работает даже с малым количеством данных
- GAN: min_G max_D [log D(x) + log(1−D(G(z)))] — генератор обманывает дискриминатор

**Связь с собесами:** VAE и GAN спрашивают на DL-собесах. Transfer learning — фундамент fine-tuning LLM (LoRA, QLoRA). KL-дивергенция — в формулах RLHF.

**Длительность:** ~85 мин | **Приоритет:** 🟢 **КРИТИЧЕСКАЯ**

Word2Vec (Skip-gram, CBOW), negative sampling, контекстные окна, GloVe, t-SNE для визуализации, Node2Vec / DeepWalk для графов.

| Временной блок | Содержание | Статус | Для собесов |
|----------------|-----------|--------|-------------|
| 0:00–0:15 | ⭐ **Дистрибутивная гипотеза: «слово определяется контекстом»** | 🟢 Ключевое | Фундамент всех word embeddings |
| 0:15–0:35 | ⭐ **Word2Vec: Skip-gram и CBOW, архитектура, обучение** | 🟢 Ключевое | «Как работает Word2Vec?» — стандартный NLP-вопрос |
| 0:35–0:50 | ⭐ **Negative sampling: зачем, как, связь с PMI** | 🟢 Ключевое | Negative sampling = ключевая оптимизация |
| 0:50–1:05 | Свойства эмбеддингов, аналогии (king−man+woman=queen) | 🟢 Смотреть | Знаменитый пример на собесах |
| 1:05–1:15 | ⭐ **t-SNE: визуализация высокомерных данных, perplexity** | 🟢 Смотреть | t-SNE vs UMAP — спрашивают |
| 1:15–1:25 | Node2Vec, DeepWalk: эмбеддинги вершин графа | 🟡 Полезно | GNN — растущая область |

**⭐ Главные инсайты:**
- Дистрибутивная гипотеза (Фёрт, 1957): «You shall know a word by the company it keeps»
- Skip-gram: по слову предсказать контекст; CBOW: по контексту предсказать слово
- Negative sampling: вместо softmax по всему словарю — бинарная классификация (слово из контекста vs случайное)
- Word2Vec ≈ факторизация PMI-матрицы (связь с матричными разложениями)
- t-SNE: сохраняет локальную структуру при проекции из высокой размерности → 2D

**Связь с собесами:** Word2Vec — один из первых вопросов на NLP-собесе. Понимание эмбеддингов обязательно для работы с Transformer и LLM.

### Лекция 4: Attention, Self-Attention, Transformer ⭐⭐⭐

**Длительность:** ~90 мин | **Приоритет:** 🟢 **КРИТИЧЕСКАЯ — САМАЯ ВАЖНАЯ ЛЕКЦИЯ КУРСА**

Seq2Seq, механизм внимания (Bahdanau), Self-Attention, Multi-Head Attention, Positional Encoding, архитектура Transformer (Attention Is All You Need), BERT, GPT.

| Временной блок | Содержание | Статус | Для собесов |
|----------------|-----------|--------|-------------|
| 0:00–0:15 | Seq2Seq с RNN, encoder-decoder, проблемы | 🟢 Смотреть | Предыстория Transformer |
| 0:15–0:30 | ⭐ **Механизм внимания (Bahdanau): Q, K, V, alignment** | 🟢 Ключевое | Attention = основа всего |
| 0:30–0:45 | ⭐ **Self-Attention: scaled dot-product, √d_k** | 🟢 Ключевое | «Почему делим на √d_k?» — частый вопрос |
| 0:45–1:00 | ⭐ **Multi-Head Attention: параллельные головы, конкатенация** | 🟢 Ключевое | «Зачем несколько голов?» |
| 1:00–1:15 | ⭐ **Transformer: Positional Encoding, FFN, Layer Norm, residuals** | 🟢 Ключевое | Полная архитектура Transformer |
| 1:15–1:30 | ⭐ **BERT (encoder), GPT (decoder), масштабирование** | 🟢 Ключевое | BERT vs GPT — стандартный NLP-вопрос |

**⭐ Главные инсайты:**
- Attention(Q,K,V) = softmax(QK^T / √d_k) · V — три матрицы: Query, Key, Value
- √d_k нужен чтобы softmax не «схлопнулся» — при больших d_k скалярные произведения растут → softmax становится almost one-hot
- Multi-Head: разные головы «смотрят» на разные типы зависимостей (синтаксис, семантика, позиция)
- Positional Encoding: sin/cos с разными частотами → даёт позиционную информацию, потому что Attention сам по себе permutation-invariant
- Transformer block = Multi-Head Attention + LayerNorm + FFN + LayerNorm + residuals
- BERT = encoder-only (bidirectional, MLM) → classification, NER
- GPT = decoder-only (causal, autoregressive) → generation, чат

**Связь с собесами:** Это **THE лекция** для NLP/LLM MLE. Attention + Transformer — спрашивают в 100% NLP-собесов. Формула scaled dot-product, Multi-Head, positional encoding, архитектурные варианты (encoder vs decoder) — всё обязательно.
