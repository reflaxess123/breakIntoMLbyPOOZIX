**Что доучить по распределениям:**

❌ **Гамма-распределение.** Обобщение экспоненциального. Сумма n экспоненциальных = Gamma(n, λ). Связь с χ² (χ² = Gamma(k/2, 2)). Не критично для собесов, но полезно для понимания.

⚠️ **Beta-распределение — углубить.** Знать: conjugate prior для Bernoulli/Binomial в байесовском выводе. Beta(α, β) → posterior после наблюдения данных. Применение: A/B-тестирование (Bayesian approach), Thompson Sampling. 🎯

### A3. Совместные распределения ❌ → нужно выучить

Это единственная серьёзная дыра в блоке A. Формально ты работаешь только с одномерными распределениями, а ML работает с многомерными.

| Тема | Статус | Приоритет |
|------|--------|-----------|
| Совместная PMF / PDF: P(X=x, Y=y), f(x,y) | ❌ | Высокий |
| Маргинальное распределение: P(X=x) = Σ_y P(X=x, Y=y) | ❌ | Высокий |
| Условное распределение: P(Y\|X) = P(X,Y) / P(X) | ❌ | Высокий |
| Условное мат.ожидание: E(Y\|X) | ❌ | Высокий 🎯 |
| Закон полного мат.ожидания: E(Y) = E(E(Y\|X)) | ❌ | Средний |
| Закон полной дисперсии: Var(Y) = E(Var(Y\|X)) + Var(E(Y\|X)) | ❌ | Средний 🎯 |
| Независимость СВ: P(X,Y) = P(X)·P(Y) | ⚠️ | Знаешь для событий, формализуй для СВ |
| Многомерное нормальное распределение | ❌ | Средний |

**Зачем это в ML:**
- E(Y|X) — это буквально то, что предсказывает регрессия
- Закон полной дисперсии — это фундамент Bias-Variance Tradeoff (вот почему у тебя пробел в формальном выводе)
- Многомерное нормальное — нужно для PCA, Gaussian Mixture Models, понимания ковариационной матрицы

**Как учить:** Ширяев "Вероятность-1" §3–4 (ты уже читаешь §1–2). Или Khan Academy "Joint distributions" — быстрее.

**Время:** ~3–4 дня.

### A4. Теория информации ⚠️

| Тема | Статус | Комментарий |
|------|--------|-------------|
| Энтропия Шеннона H(P) | ✅ | — |
| Cross-entropy H(P,Q) | ✅ | Полный вывод из MLE |
| Связь MLE → CE | ✅ | Глубоко |
| KL-дивергенция D_KL(P‖Q) | ⚠️ | Знаешь H(P,Q) = H(P) + D_KL, но не разбирал формально |
| Взаимная информация I(X;Y) | ❌ | — |

**Что доучить:**

⚠️ **KL-дивергенция — формализовать.** Ты уже знаешь главное (связь с CE), осталось:
- Формула: D_KL(P‖Q) = Σ P(x) · log(P(x)/Q(x))
- Асимметричность: D_KL(P‖Q) ≠ D_KL(Q‖P) — и почему это важно
- Forward KL vs Reverse KL: forward (mode-covering) vs reverse (mode-seeking) — это спрашивают на собесах в контексте VAE и distillation 🎯
- Неотрицательность: D_KL ≥ 0, равенство только при P = Q (через неравенство Йенсена)
- Связь с cross-entropy: минимизировать CE ↔ минимизировать KL (т.к. H(P) = const)

❌ **Взаимная информация I(X;Y).** Формула: I(X;Y) = H(X) − H(X|Y) = D_KL(P(X,Y) ‖ P(X)·P(Y)). Применение в ML: feature selection (mutual information между фичей и таргетом). 🎯 Спрашивают в контексте feature importance.

**Время:** ~1–2 дня (KL ты почти знаешь, надо дожать).

### B2. Bias-Variance Tradeoff ⚠️ → формализовать

| Тема | Статус |
|------|--------|
| Концепт: underfitting = high bias, overfitting = high variance | ✅ |
| Регуляризация добавляет bias, снижает variance | ✅ |
| Формальный вывод MSE = Bias² + Variance + Noise | ❌ 🎯 |

**Что конкретно вывести:**

Для предсказания f̂(x) обученной модели при фиксированном x:

```
MSE = E[(y − f̂(x))²]

Раскладывается:

MSE = (E[f̂(x)] − f(x))²  +  Var(f̂(x))  +  σ²
       ─────────────────     ─────────────     ──
            Bias²              Variance       Irreducible noise
```

**Ключ к выводу** — тебе нужен **закон полной дисперсии** из A3. Вот почему совместные распределения — prerequisite.

Шаги вывода:
1. y = f(x) + ε, где ε ~ N(0, σ²)
2. MSE = E[(y − f̂)²] = E[(f + ε − f̂)²]
3. Раскрываем квадрат, используем E(ε) = 0, E(ε²) = σ²
4. Добавляем и вычитаем E[f̂]: (f − f̂) = (f − E[f̂]) + (E[f̂] − f̂)
5. Квадрат суммы → Bias² + Variance + cross-term (= 0) + σ²

**Приоритет: ВЫСОКИЙ** 🎯 — один из самых частых теоретических вопросов на собесах.

**Время:** ~1 день (если A3 уже закрыта).

### C2. Параметрические тесты ✅ / ⚠️

| Тест | Статус | Что тестирует |
|------|--------|---------------|
| Z-тест (одновыборочный) | ✅ | μ = μ₀ при известной σ |
| Z-тест (двухвыборочный) | ✅ | μ₁ = μ₂ при известных σ |
| t-тест (одновыборочный) | ✅ | μ = μ₀ при неизвестной σ |
| t-тест (двухвыборочный, равные дисп.) | ✅ | μ₁ = μ₂ |
| t-тест Уэлча (неравные дисп.) | ⚠️ | μ₁ = μ₂ без допущения σ₁=σ₂ |
| Парный t-тест | ⚠️ | Зависимые выборки (до/после) |
| **Хи-квадрат тест (goodness of fit)** | ❌ 🎯 | Данные соответствуют распределению? |
| **Хи-квадрат тест (независимости)** | ❌ 🎯 | Две категориальные переменные независимы? |
| **F-тест** | ❌ | σ₁² = σ₂² |
| **ANOVA (одностороннее)** | ❌ 🎯 | μ₁ = μ₂ = ... = μₖ (3+ группы) |

**Что доучить:**

❌ **Хи-квадрат тест** — два применения:

**1. Goodness of fit** (согласие): наблюдаемые частоты vs ожидаемые.
- Статистика: χ² = Σ (Oᵢ − Eᵢ)² / Eᵢ
- df = (число категорий − 1)
- Пример: "монета честная?" → O = [53, 47], E = [50, 50], χ² = 0.36, df=1, p = 0.55 → не отвергаем H₀
- 🎯 Собесный контекст: проверка распределения фичей, проверка рандомизации в A/B

**2. Тест независимости** (contingency table):
- Таблица наблюдений R×C, ожидаемые = (сумма строки × сумма столбца) / N
- df = (R−1)(C−1)
- 🎯 Собесный контекст: "есть ли связь между полом и кликом на баннер?"

❌ **F-тест и ANOVA:**

**F-тест** — сравнение двух дисперсий: F = s₁²/s₂² ~ F(df₁, df₂). Используется как building block для ANOVA.

**ANOVA (Analysis of Variance)** — обобщение t-теста на 3+ группы:
- H₀: μ₁ = μ₂ = ... = μₖ
- Идея: сравниваем вариацию *между* группами с вариацией *внутри* групп
- F = MS_between / MS_within
- Если F большое → группы различаются
- 🎯 Собесный контекст: "3 варианта дизайна, какой лучше?" — нельзя просто делать 3 попарных t-теста (проблема множественных сравнений)

**Время:** ~3–4 дня (хи-квадрат + F-тест + ANOVA).

### C3. Непараметрические тесты ✅ / ❌

| Тест | Статус | Что тестирует | Параметрический аналог |
|------|--------|---------------|----------------------|
| Вилкоксон (signed-rank) | ✅ | Медиана = m₀ | Одновыборочный t-тест |
| Манн-Уитни U | ❌ | Две независимые выборки | Двухвыборочный t-тест |
| Критерий знаков | ❌ | Медиана (проще Вилкоксона) | — |
| Колмогоров-Смирнов | ❌ | CDF двух выборок совпадают? | — |
| Перестановочный тест (permutation) | ❌ 🎯 | Любая разница между группами | t-тест |

**Что доучить:**

❌ **Манн-Уитни U** — непараметрический аналог двухвыборочного t-теста. Ранжируем все наблюдения вместе, сравниваем суммы рангов. Не требует нормальности. 🎯 Нужен для A/B когда метрика не нормальная.

❌ **Перестановочный тест** (permutation test) — самый интуитивный:
1. Считаем разницу средних между A и B
2. Перемешиваем метки A/B случайно, считаем разницу снова
3. Повторяем 10000 раз → получаем распределение "при H₀"
4. Смотрим, насколько экстремальна наблюдаемая разница
- 🎯 Часто используется в A/B (bootstrap + permutation — два главных resampling-метода)
- Связь с bootstrap: bootstrap оценивает CI для одной статистики, permutation тестирует H₀ о различии

❌ **Колмогоров-Смирнов** — сравнение CDF. KS-статистика = max|F₁(x) − F₂(x)|. Используется для проверки нормальности данных. **Приоритет: средний.**

**Время:** ~2 дня.

### C4. Мощность и размер выборки ❌

| Тема | Статус | Приоритет |
|------|--------|-----------|
| Мощность теста (power = 1 − β) | ❌ | Высокий 🎯 |
| Связь α, β, n, effect size | ❌ | Высокий 🎯 |
| Power analysis (расчёт n до эксперимента) | ❌ | Высокий 🎯 |
| Effect size (Cohen's d, Cohen's h) | ❌ | Средний |
| MDE (Minimum Detectable Effect) | ❌ | Высокий 🎯 |

**Зачем:** Без этого невозможно грамотно спланировать A/B-тест. Это prerequisite для блока D.

**Ключевые формулы:**

Power = P(отвергнуть H₀ | H₁ верна). Зависит от четырёх вещей:
- **α** (уровень значимости) — обычно 0.05
- **Effect size** (насколько большой реальный эффект) — Cohen's d = (μ₁−μ₂)/σ
- **n** (размер выборки) — чем больше, тем выше power
- **Дисперсия** σ² — чем меньше шум, тем легче обнаружить эффект

Стандартная цель: power ≥ 0.8 (80%).

**Формула размера выборки для двухвыборочного z-теста:**

```
n = 2 · ((z_{α/2} + z_β) / d)²

где d = (μ₁ − μ₂) / σ — effect size (Cohen's d)
    z_{α/2} ≈ 1.96 (для α = 0.05)
    z_β ≈ 0.84 (для power = 0.80)
```

🎯 **Собесный вопрос:** "Как рассчитать, сколько пользователей нужно для A/B-теста?" → Power analysis с заданным MDE.

**Время:** ~2 дня.

### C5. Множественное тестирование ❌

| Тема | Статус | Приоритет |
|------|--------|-----------|
| Проблема множественных сравнений | ❌ | Высокий 🎯 |
| Поправка Бонферрони (α/m) | ❌ | Высокий 🎯 |
| Benjamini-Hochberg (FDR) | ❌ | Средний |
| Family-wise error rate (FWER) | ❌ | Средний |

**Суть проблемы:** Если тестируешь 20 метрик с α=0.05, в среднем 1 покажет "значимость" чисто случайно. Это называется multiple comparisons problem.

**Бонферрони:** Простейшее решение — делим α на число тестов m. Если 20 метрик, используем α' = 0.05/20 = 0.0025 для каждого. Консервативно, но надёжно.

**Benjamini-Hochberg (FDR):** Менее консервативный. Контролирует долю ложных открытий среди всех "значимых" результатов. Более мощный, чем Бонферрони.

🎯 **Собесный вопрос:** "Вы запустили A/B-тест и смотрите 15 метрик. 3 показали p < 0.05. Что вы скажете менеджеру?"

**Время:** ~1 день.

| Выбор метрики (primary, secondary, guardrail) | ❌ | Критический 🎯 |
| Рандомизация (user-level, session-level) | ❌ | Высокий |
| Расчёт sample size (power analysis) | ❌ | Критический 🎯 |
| Продолжительность эксперимента | ❌ | Высокий |
| Анализ результатов (t-тест, z-тест для пропорций) | ❌ | Критический 🎯 |
| Peeking problem (ранняя остановка) | ❌ | Высокий 🎯 |
| Sequential testing | ❌ | Средний |
| CUPED (variance reduction) | ❌ | Средний |
| Множественное тестирование в A/B | ❌ | Высокий 🎯 |
| Сегментация (novelty effect, Simpson's paradox) | ❌ | Средний |
| Bayesian A/B (Beta-Binomial) | ❌ | Средний |

**Как учить A/B — по шагам:**

```
Шаг 1: Формулируем гипотезу
   H₀: conversion_A = conversion_B
   H₁: conversion_A ≠ conversion_B

Шаг 2: Выбираем метрики
   Primary: CTR (click-through rate)
   Guardrail: revenue per user не упал

Шаг 3: Считаем sample size
   MDE = 2% абсолютного увеличения CTR
   Baseline CTR = 10%
   α = 0.05, power = 0.8
   → n ≈ 3900 на группу (формула для пропорций)

Шаг 4: Запускаем, ждём
   Нельзя подглядывать (peeking → inflated α)

Шаг 5: Анализируем
   z-тест для пропорций или t-тест для средних
   Проверяем множественные сравнения если метрик > 1
```

**Время:** ~4–5 дней (это большая тема).

### D2. Bootstrap ✅ → дорасширить

| Тема | Статус |
|------|--------|
| Принцип (ресемплинг с возвращением) | ✅ |
| Bootstrap CI | ✅ |
| OOB | ✅ |
| Percentile bootstrap vs BCa | ⚠️ |
| Bootstrap для A/B | ❌ |

**Что доучить:**

⚠️ **Типы bootstrap CI:**
- Percentile method: просто берём [2.5%, 97.5%] от bootstrap-распределения. Прост, но bias-prone.
- BCa (bias-corrected and accelerated): корректирует за смещение и асимметрию. Стандарт в scipy/statsmodels.

❌ **Bootstrap в A/B:** вместо t-теста можно бутстрапить разницу средних → CI для (μ_B − μ_A). Если CI не содержит 0 → значимо. Преимущество: работает для любой метрики (медиана, 95-й перцентиль, ratio метрики).

**Время:** ~1 день.

### D3. Метрики ML ❌ → нужно выучить

#### Метрики классификации

| Метрика | Статус | Формула | Приоритет |
|---------|--------|---------|-----------|
| Confusion Matrix | ❌ | TP, TN, FP, FN | Критический 🎯 |
| Accuracy | ❌ | (TP+TN) / (TP+TN+FP+FN) | Критический |
| Precision | ❌ | TP / (TP+FP) — "из предсказанных + сколько верных" | Критический 🎯 |
| Recall (Sensitivity) | ❌ | TP / (TP+FN) — "из реальных + сколько нашли" | Критический 🎯 |
| F1-score | ❌ | 2·P·R / (P+R) — гармоническое среднее | Критический 🎯 |
| Specificity | ❌ | TN / (TN+FP) | Средний |
| AUC-ROC | ❌ | Площадь под ROC-кривой (TPR vs FPR) | Критический 🎯 |
| AUC-PR | ❌ | Площадь под Precision-Recall кривой | Высокий |
| Log Loss | ❌ | −Σ y·log(ŷ) (= cross-entropy, это ты знаешь!) | Средний |

🎯 **Ключевой собесный вопрос:** "Precision или Recall — когда что важнее?"
- **Precision важнее** когда false positive дорогой: спам-фильтр (нельзя важное письмо в спам), рекомендации (нельзя показывать нерелевантное)
- **Recall важнее** когда false negative дорогой: диагностика рака (нельзя пропустить больного), fraud detection (нельзя пропустить мошенника)

🎯 **Ещё вопрос:** "Accuracy 95% — это хорошо?" → Нет, если класс 1 = 5% данных (при always predict 0 accuracy = 95%). Используй F1 или AUC для imbalanced datasets.

#### Метрики регрессии

| Метрика | Статус | Формула | Приоритет |
|---------|--------|---------|-----------|
| MSE | ⚠️ | Σ(y−ŷ)²/n — знаешь как loss, не как метрику | Высокий |
| RMSE | ❌ | √MSE — в единицах таргета | Высокий 🎯 |
| MAE | ❌ | Σ\|y−ŷ\|/n — робастнее к выбросам | Высокий 🎯 |
| R² | ❌ | 1 − SS_res/SS_tot — доля объяснённой дисперсии | Высокий 🎯 |
| MAPE | ❌ | Σ\|y−ŷ\|/y · 100% — процентная ошибка | Средний |

🎯 **Собесный вопрос:** "RMSE vs MAE — когда что?" → RMSE сильнее штрафует за большие ошибки (из-за квадрата). Если выбросы — проблема, MAE лучше.

🎯 **R² — что значит?** R² = 0.85 → модель объясняет 85% вариации таргета. R² = 0 → модель не лучше предсказания средним. R² < 0 → модель хуже среднего.

**Время:** ~2–3 дня.
